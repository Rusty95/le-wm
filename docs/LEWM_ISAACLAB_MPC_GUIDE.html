1. 当前系统一句话

它不是直接让网络输出 action，而是让 LeWM 预测未来 latent，再用 MPC 搜索动作。

在线闭环

每个 IsaacLab step 都重新拿相机图像，重新规划动作，只执行最优动作序列的第一步。

pixels, state = env.observe()
action = mpc_plan(pixels, state)
env.step(action)

LeWM 的角色

LeWM 不是 policy。它负责回答：如果接下来执行这串动作，未来 latent 会变成什么样。

pixels -> encoder -> emb
actions -> action_encoder -> act_emb
emb + act_emb -> predictor -> pred_emb
当前最重要的结论：这套系统已经能在接近平衡的初始状态下接近稳定控制，曾跑到 299 / 300 steps；但初始 pole 角达到 0.32~0.38 rad 时，恢复能力仍不足。

0.064最佳单条 mean cart
0.013最佳单条 mean pole angle

4. Cost 与控制技巧

当前控制效果主要来自三个 cost 的组合。

latent-target：让未来接近稳定画面

从稳定数据里挑选接近 upright / centered 的帧，编码成 target_emb，然后让候选动作预测出来的 pred_emb 接近它。

cost_latent = distance(pred_emb, target_emb)

优点是直接使用 LeWM 的 latent 空间，不依赖 decoder。缺点是它不天然知道“该往哪边推”。

action prior：给 MPC 一个局部救杆方向

根据当前真实 state 计算一个连续动作先验。

score = kp_pole*pole_pos + kd_pole*pole_vel + kp_cart*cart_pos + kd_cart*cart_vel
a_prior = -score
cost_prior = weight * (action[0] - a_prior)^2

它像一个小的 PD / LQR 提示器，让 CEM 不完全靠 latent cost 摸索方向。

edge rescue：小车接近边界时接管

当小车靠近边界且还在向外运动时，鼓励动作往中间拉，同时压低 action prior 的权重。

activation = relu(|cart_pos|-threshold) * (1 + outward_vel)
target_action = -sign(cart_pos) * return_action
effective_prior_weight = prior_weight * (1 - edge_gate)

这个技巧把失败模式从“cart 出界”转成了“pole 后期发散”，说明它确实解决了边界问题的一部分。

CEM：从很多动作序列里筛精英

每一步采样很多候选动作序列，预测未来 latent，计算 cost，选 elite 更新分布。最终只执行第一步。

samples -> rollout -> cost -> top elite -> update mean/std -> repeat

horizon=4 目前最好；horizon=6 反而变差，说明 LeWM 多步预测误差会影响规划。

5. 参数案例

这些是我们已经跑过并分析过的典型组合。

方案核心参数结果特征适用判断
h4 baseline horizon=4, kp=2.4, kd=0.6, edge_return=1.0 平均 221/300，出现过 299/300，cart 最居中。 当前最平衡的方案。
pole30_kd08 kp=3.0, kd=0.8 平均 survival 更高一点，但 cart 更偏。 更适合救较大的初始 pole 角。
h6 baseline horizon=6 平均变差，说明预测更远不一定更好。 不推荐作为默认。
return06 edge_return=0.6 回中变温和，但 survival 和 cart 都变差。 已基本排除。
direction_bias 符号方向强救 杆角很小，但动作高频抖动，小车接近出界。 适合做对照，不适合作最终方案。
重要现象：同样参数下，初始 pole 角差别很大时结果会差很多。接近 0 rad 时可以接近满分；初始 0.32~0.38 rad 时容易只撑 120 步左右。

6. 怎么看结果

不要只看 reward，要看失败原因。

核心指标

survival_steps 是否接近 300。

mean_abs_pole_angle 杆是否稳。

mean_abs_cart_pos 小车是否居中。

max_cart 是否接近边界。

edge_align edge 触发后动作是否朝 target 方向。

典型失败模式

cart 出界：需要 edge rescue 更早或更强。

pole 爆掉：需要更强 pole prior、更多大角度数据，或更好的 future risk cost。

动作抖动：direction-bias 太强或 action smoothness 太弱。

horizon 变长反而差：LeWM 多步 latent rollout 误差太大。

8. 下一步建议

现在最值得做的是补足大角度恢复能力。

短期调参

针对初始大倾角，试 prior_pole_kp=3.0、prior_pole_kd=0.8，观察是否能救回 0.3 rad 以上的初始角。

数据增强

采集大倾角恢复数据，尤其是 pole angle 大、pole velocity 大时 PPO 如何救杆。

模型增强

增加 termination risk 或 future pole risk 的预测头，让 MPC 不只看 latent target。

当前边界：这还不是纯视觉控制。当前 action prior 和 edge rescue 使用真实 state。后续如果要更“世界模型味”，需要把这些 state 估计也迁到视觉 latent / probe 上。
